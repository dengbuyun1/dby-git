# 人工胰腺闭环控制系统：EKF 与 Smith 预估器核心模块原理解析

本文档详细梳理了当前项目中用于处理**传感器噪声**、**系统状态估计**以及**网络与生理延迟**的两大核心模块：**扩展卡尔曼滤波 (EKF)** 与 **Smith 预估器**的工程实现结构与原理。

在人工胰腺（血糖闭环控制）系统中，我们面临两个巨大的挑战：
1. **看不见全貌且有误差**：连续血糖监测仪 (CGM) 只能测量皮下组织间液的葡萄糖浓度，且伴有测量噪声。而控制器需要知道体内真实的胰岛素作用状态才能精准给药。
2. **时间差 (纯滞后)**：胰岛素从泵入皮下到真正发挥降糖作用存在生理延迟，同时无线网络传输也存在通信延迟。如果控制器基于“过去”的血糖状态计算“现在”的给药量，极易导致系统震荡甚至低血糖。

EKF 和 Smith 预估器正是为了解决这两个致命问题而设计的。

---

## 一、 扩展卡尔曼滤波 (EKF)：系统的“透视眼”与“测谎仪”

**一句话理解**：EKF 就像一个经验丰富的老中医，它不仅“听”（读取有噪声的 CGM 数据），还结合自己脑海中的“医学理论”（生理学数学模型），综合推断出患者当前体内真实的、看不见的生理状态（如真实的血糖浓度、胰岛素活性）。

### 1. 核心实现结构

我们的 EKF 模块 (`src/estimation/ekf.py`) 构建在一个精简而高效的底层生理模型之上：

*   **底层引擎：Bergman 最小模型 (`src/model/physiology.py`)**
    为了保证实时计算的高效性，系统没有采用包含几十个参数的臃肿模型，而是使用了经典的 Bergman 最小模型。该模型用三个核心变量描述人体状态：
    *   `G`: 真实的血浆葡萄糖浓度 (Plasma Glucose)
    *   `X`: 胰岛素的降糖作用活性 (Insulin Action)
    *   `I`: 血浆胰岛素水平代理变量 (Plasma Insulin Proxy)

*   **预测步 (Predict)：“理论推演”**
    在每个控制周期，EKF 利用基础的生理模型，结合刚才注入的胰岛素量和进食情况，**预测**当前人体状态应该演变到了什么地步。

*   **更新步 (Update)：“现实校准”**
    随后，EKF 读取最新传来的 CGM 血糖值。将“理论预测的血糖”与“实际测量的血糖”进行比对，利用精密的矩阵运算（卡尔曼增益 $K$），自动平衡“对模型的信任度”和“对传感器的信任度”，从而得出一个最接近真相的状态估计值 $\hat{x}$。

### 2. 工程亮点与优势

*   **解析法雅可比矩阵 (Analytical Jacobian)**：在非线性系统中，EKF 需要计算偏导数矩阵（雅可比矩阵）。我们的代码没有使用粗糙且耗时的数值差分法，而是直接在底层模型中手写了精确的微分解析式（`jacobian_f` 和 `jacobian_h`），这使得算法极度轻量，非常适合部署在算力有限的树莓派等嵌入式设备上。
*   **不仅是透视眼，更是“测谎仪”（故障诊断基石）**：
    EKF 每次更新不仅输出状态，还会输出一个极其关键的指标：**新息 (Innovation) / 残差**（即预测血糖与实测血糖的差值）。如果泵发生了堵塞或电机故障，模型预测血糖应该下降，但实测血糖却居高不下，此时残差会瞬间飙升。结合代码中输出的 **NIS（归一化新息平方）** 指标，我们的故障诊断模块 (`fault_detector.py`) 可以极其灵敏且无误报地捕捉到硬件异常。

## 二、 Smith 预估器：战胜“时间差”的时间机器

**一句话理解**：既然我们看到的 CGM 数据是延迟了 $T$ 分钟之前的状态，Smith 预估器就像一台“时间机器”，它利用我们已知的这 $T$ 分钟内打过的胰岛素记录，把那个“过去的残影”加速快进推演到现在，给控制器提供一个“假装没有延迟”的即时反馈。

### 1. 核心实现结构

我们的 Smith 预估器 (`src/compensation/smith.py`) 与 EKF 深度联动，构成了一个完美的防延迟闭环：

*   **历史行为记录本 (`insulin_history_u_per_min`)**
    在主控循环 (`orchestrator.py`) 中，系统会精准记录过去一段时间（长度等于预估的延迟步数）内每一分钟泵入的实际胰岛素剂量。

*   **状态前向推演 (`predict_for_control`)**
    由于网络传输等原因，EKF 算出的最新状态 $\hat{x}$ 实际上也是滞后的。Smith 预估器接手这个滞后状态，利用底层生理模型 (`PhysiologyModel`)，并喂入这期间的历史胰岛素记录，在纯数字世界里将状态**一步步推演到真正的“现在时刻”**。

*   **构造“无延迟”虚拟反馈 (`compensate_signal`)**
    这是 Smith 预估器最核心的魔法。它并不直接把推演出的未来状态丢给控制器（因为纯理论推演可能有误差），而是巧妙地计算一个**修正值**，公式如下：
    $$G_{corrected} = G_{measured} + (G_{pred\_now} - G_{hat\_delayed})$$
    *(修正后的即时血糖 = 实测的滞后血糖 + (模型推演的当前血糖 - 模型推演的滞后血糖))*
    
    这个 $G_{corrected}$ 完美抵消了纯滞后时间，被直接送入 PID 控制器中。控制器看到的是一个响应极其迅速的系统，从而敢于更加积极地调节，避免了因延迟导致的过量给药和低血糖震荡。

### 2. 工程亮点与优势

*   **动态延迟追踪 (`DelayStepEstimator`)**：
    传统的 Smith 预估器最大的弱点是“必须准确知道延迟时间，且延迟必须固定”。
    我们的工程实现中 (`src/compensation/delay_tracker.py`) 创新性地引入了**自适应延迟追踪**。它利用底层 TCP 通讯每次请求返回的往返时间 (RTT)，通过指数移动平均 (EMA) 算法，实时、动态地估算出当前网络和系统的真实延迟步数。这意味着无论网络如何抖动，Smith 预估器都能自我调整，始终提供最精准的补偿。

## 总结：黄金搭档

在我们的架构中：
1. **EKF** 负责剔除传感器噪声，看穿人体内部状态，并兼职硬件故障的“报警器”。
2. **Smith 预估器** 站在 EKF 的肩膀上，利用历史记录消除时间延迟的魔咒。

这两者结合，为最外层的闭环控制算法（无论是 PID 还是 MPC）提供了一个干净、即时、且包含内部状态深度的“完美虚拟患者接口”，极大地提升了整套人工胰腺系统的鲁棒性与安全性。
