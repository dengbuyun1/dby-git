# 项目阶段性进展汇报：人工胰腺与数字孪生闭环控制系统

本报告详细梳理了当前 `mx` 工程目录下各项核心工作的完成情况、工程亮点、取得的成果，以及面临的问题与下一步的优化方向。**本报告专为撰写 PPT 汇报材料设计。**

---

## 一、 已完成的工作 (Work Completed)

项目组对系统的控制算法后端进行了深度的重构与升级，成功落地了针对传感器噪声、系统延迟以及硬件故障的三大核心模块，并打通了数字孪生在线推断的链路。

### 1. 扩展卡尔曼滤波 (EKF) 状态观测器
**目标：** 过滤 CGM 传感器噪声，推断患者体内不可观测的真实生理状态（血糖、胰岛素活性）。
*   **代码实现：** 位于 `src/estimation/ekf.py` 及 `src/model/physiology.py`。
*   **核心逻辑：**
    *   **底层生理模型**：采用经典的 Bergman 最小模型 (Minimal Model)。
        *   状态变量 $x = [G, X, I]^T$ (分别代表真实血糖、胰岛素活性、血浆胰岛素水平)。
        *   连续状态微分方程：
            $$ \dot{G} = -p_1(G-G_b) - X \cdot G + Meal $$
            $$ \dot{X} = -p_2 X + p_3(I - I_b) $$
            $$ \dot{I} = -n(I - I_b) + \frac{U}{V_i} $$
    *   **观测模型**：$z = Hx = [1, 0, 0] \cdot [G, X, I]^T = G$ (仅血糖可观测)。
    *   **滤波过程**：实现了标准的非线性预测 (Predict) 和线性化更新 (Update) 步骤，计算卡尔曼增益 $K$，不断修正状态估计值 $\hat{x}$。

### 2. Smith 预估器 (延迟补偿模块)
**目标：** 克服网络通讯和胰岛素生理吸收带来的纯滞后时间 ($T_d$)，防止闭环控制产生震荡。
*   **代码实现：** 位于 `src/compensation/smith.py`。
*   **核心逻辑：**
    *   系统在主循环中维护一个长度为延迟步数的输入历史队列 `insulin_history_u_per_min`。
    *   **状态前向推演**：利用底层的 Bergman 模型，将 EKF 估算出的滞后状态 $\hat{x}_{delayed}$，喂入历史输入，一步步前向推演到“现在的真实时刻”得到 $G_{pred\_now}$。
    *   **虚拟反馈修正公式**：构造无延迟的反馈信号输入给 PID/MPC 控制器：
        $$ G_{corrected} = G_{measured} + (G_{pred\_now} - G_{hat\_delayed}) $$

### 3. 基于残差的执行器故障检测 (Fault Detection)
**目标：** 实时监控下位机（如胰岛素泵电机堵转、管路堵塞）和传感器的硬件异常。
*   **代码实现：** 位于 `src/estimation/fault_detector.py`。
*   **核心逻辑：**
    *   提取 EKF 滤波过程中的**新息 (Innovation / Residual)**：$r = z_{measured} - z_{predicted}$。
    *   提取**归一化新息平方 (NIS)** 指标：$NIS = r^T S^{-1} r$ (其中 S 为新息协方差矩阵)。
    *   设定了严密的逻辑防抖状态机：当绝对残差或 NIS 超过安全阈值，且连续触发次数达到 `trigger_count` 时，锁死并触发安全报警，切断控制回路。

### 4. SBI (基于仿真的推断) 数据链路集成
**目标：** 利用深度学习（前馈神经网络如 NPE）在线更新数字孪生的个性化参数。
*   **代码实现：** 位于 `src/sbi_simglucose/` 及 `scripts/` 目录。
*   **完成情况：** 跑通了工程主链路。闭环调度器 (`ClosedLoopOrchestrator`) 能够按固定周期调用 `sbi_engine` 抓取近期 CGM 序列，推断出针对 Bergman 模型中 $[k_{abs}, k_{p1}, k_{p2}, k_{p3}]$ 等参数的动态缩放因子，实现在线自适应。

## 二、 完成情况与工程亮点 (Highlights)

1.  **架构闭环，高度内聚**：成功将 EKF 观测、Smith 补偿、故障判定、SBI 推断统一集成到了一个调度器类 (`ClosedLoopOrchestrator`) 中，数据流转清晰，无强耦合。
2.  **放弃数值差分，引入解析法雅可比矩阵**：在 `physiology.py` 中手写推导了非线性偏导数矩阵 (`jacobian_f`, `jacobian_h`)。这极大降低了 EKF 的运算开销，为后续移植到算力极弱的树莓派单片机扫清了障碍。
3.  **动态自适应延迟追踪技术**：编写了 `DelayStepEstimator` 模块。系统没有死板地设定固定的网络延迟时间，而是通过 TCP 通讯返回的真实 RTT (往返时间)，运用 EMA (指数移动平均) 算法，**实时动态地计算当前应该补偿的延迟步数**，极大增强了 Smith 预估器在恶劣网络下的鲁棒性。
4.  **科学的防抖报警机制**：故障诊断模块不仅仅依靠单次误差绝对值，而是引入了结合模型置信度矩阵 $P$ 的 **NIS 检测**，并辅以连续触发 (`trigger_count`) 和连续恢复 (`clear_count`) 的状态机，从根本上杜绝了 CGM 偶发噪声引起的假阳性停机报警。

## 三、 取得的成果 (Achievements)

1.  建立了一个临床可用级别、抗网络/生理双重延迟、具备高抗噪能力的**安全闭环控制软件基座**。
2.  在算法代码层面，成功复现了最新学术论文中提及的基于 EKF 残差的高灵敏度闭环泵故障检测机制。
3.  通过创新的 RTT 自适应追踪结合 Smith 预估器，提供了一种解决穿戴式医疗物联网 (IoT) 实时控制延迟问题的新型工程方案。
4.  打通了 SBI 数据流水线，为下一代“数字孪生驱动的个体化人工胰腺”验证了技术可行性。

## 四、 存在的问题与局限性 (Existing Issues & Limitations)

1.  **SBI 核心模型存在“参数不可识别性” (Parameter Identifiability Problem)**：
    目前工程中 SBI 模块生成训练数据和离线推演时，依然绑定在包含数十个不可观测参数的复杂仿真模型 (如 `simglucose`) 上。数学理论证明，仅靠一维的 CGM 曲线反推如此庞大的参数空间是病态 (Ill-posed) 的，这会导致 SBI 推断出的后验分布极度平坦或不可靠，参数失去明确的生理学指导意义。
2.  **未接入 `py-mgipsim` 模型**：
    虽然原计划采用该模型，但经项目组深入分析，发现 `py-mgipsim` 本质上依然属于高度复杂的房室机制 ODE 模型，强行接入只会增加工程复杂度，无法从根本上解决上述的“参数不可识别”痛点。
3.  **数值积分的局限**：
    底层生理方程的求解目前采用的是简单的一阶欧拉积分 (`Euler Integration`)。在面对患者剧烈进食或大剂量推注胰岛素导致系统发生强非线性突变时，欧拉法可能会产生数值截断误差。

## 五、 后续计划 (Future Plans)

1.  **核心路线调整：全面转向“最小模型”进行 SBI 参数推断 (Plan A)**
    *   **行动：** 放弃在复杂的 `simglucose` 或 `py-mgipsim` 上进行参数学习。将 SBI 神经网络（NPE）的推断目标完全锚定在你手写的精简版 Bergman 最小模型上。
    *   **预期收益：** 最小模型参数极少（仅 $p_1, p_2, p_3, n$ 等），具有极高的参数可识别性。SBI 能精准收敛，输出高置信度的参数概率分布，且推断出的参数可**直接无缝喂给 EKF 和 Smith 预估器使用**，实现孪生系统与控制系统的完美统一。
2.  **算法精度优化：升级 ODE 求解器**
    *   **行动：** 修改 `src/model/physiology.py`，将积分求解算法从一阶欧拉法升级为标准的四阶龙格-库塔法 (Runge-Kutta, RK4)。
    *   **预期收益：** 以极小的计算代价，大幅提升 EKF 状态预测步在剧烈波动期的数值稳定性和预测精度。
3.  **长期硬件联调与闭环测试**
    *   **行动：** 将当前完善的 Python 上位机系统，与树莓派（下位机）、TCP 网络、真实的电机驱动模块连接，开展长时间无断点联调。
    *   **预期收益：** 在真实的物理延迟、机械摩擦阻力和真实的无线网络抖动环境中，验证 EKF 和 Smith 预估器以及故障诊断状态机的实际效能。
