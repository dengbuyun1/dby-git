# 数据库字段修复说明

## 问题描述
错误信息：`(1054, "Unknown column 'patient_id' in 'SELECT'")`

**原因**：数据库中的 `sim_meta` 表已经存在，但是缺少新增的患者信息字段。

## 已完成的修复

### 1. database.py - 自动添加缺失字段 ✅
修改了 `ensure_tables()` 函数，现在会自动检查并添加缺失的字段：
- patient_id
- patient_name
- patient_type
- patient_age
- patient_gender
- patient_blood_type

### 2. base.py - 修复JSON序列化错误 ✅
修复了 `params_updated` 响应中 datetime 对象无法序列化的问题。

## 手动修复方法（备选）

如果自动修复失败，可以手动执行SQL：

```sql
USE 2patients_datas;

-- 添加患者信息字段到 sim_meta 表
ALTER TABLE sim_meta ADD COLUMN patient_id VARCHAR(50);
ALTER TABLE sim_meta ADD COLUMN patient_name VARCHAR(100);
ALTER TABLE sim_meta ADD COLUMN patient_type VARCHAR(50);
ALTER TABLE sim_meta ADD COLUMN patient_age INT;
ALTER TABLE sim_meta ADD COLUMN patient_gender VARCHAR(20);
ALTER TABLE sim_meta ADD COLUMN patient_blood_type VARCHAR(10);

-- 查看表结构确认
DESCRIBE sim_meta;
```

## 常见问题

**Q: 仍然报错 "Unknown column"？**
A: 
1. 确认数据库连接正确
2. 使用SQL客户端手动执行ALTER TABLE语句
3. 重启后端程序

**Q: 显示 "Duplicate column name"？**
A: 这是正常的，说明字段已经存在，程序会自动忽略

**Q: JSON序列化错误？**
A: 已修复，base_datas 中的 datetime 对象已转换为 ISO 字符串
