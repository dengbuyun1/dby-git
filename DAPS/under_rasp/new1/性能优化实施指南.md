### **步骤3: 修改 main.py 的清理逻辑**

在 `main.py` 的 `finally` 块中添加:

```python
# 原 main.py
def main():
    try:
        db_connection = db.connect_mysql()
        # ... 其他代码 ...
    finally:
        if "db_connection" in locals() and db_connection:
            db_connection.close()  # 删除这行 (异步模式不需要)
        
        # 添加这行:
        db.stop_async_db()  # 停止异步写入线程
        
        # ... 其他清理 ...
```

完整修改后:

```python
def main():
    try:
        db_connection = db.connect_mysql()
        pwm = motor.setup_gpio()
        bluetooth_thread = bt.start_bluetooth_server(db_connection, db.save_to_database)
        motor.motor_control_loop(pwm, bt)

    except KeyboardInterrupt:
        print("程序中断")
    except Exception as e:
        print(f"主程序错误: {e}")
    finally:
        # 停止异步数据库 (必须!)
        db.stop_async_db()
        
        if "pwm" in locals():
            try:
                pwm.stop()
                GPIO.cleanup()
            except:
                pass
        print("程序结束")
```

## ⚠️ 常见问题

### **Q1: 数据库写入失败**

**现象**: 日志显示 "批量写入错误"

**原因**: MySQL 连接失败或权限问题

**解决**:
```bash
# 检查MySQL服务
sudo systemctl status mysql

# 检查配置
vim config_module.py  # 确认 DB_CONFIG 正确
```

---

### **Q2: 队列溢出**

**现象**: "数据库队列已满"

**原因**: 数据库写入速度 < 数据产生速度

**解决**:
```python
# 在 database_module_async.py 中调整:
AsyncDatabaseWriter(
    batch_size=20,      # 增大批大小 (默认10)
    flush_interval=0.5  # 缩短刷新间隔 (默认1.0)
)
```

---

### **Q3: 程序退出时卡住**

**现象**: Ctrl+C 后等待很久

**原因**: 异步线程在刷新剩余数据

**解决**: 正常现象,等待数据刷新完毕 (最多10秒)

---

### **Q4: 没有性能提升**

**检查清单**:
1. ✅ 确认使用了异步数据库模块
2. ✅ 确认 `db.stop_async_db()` 在 finally 中调用
3. ✅ 检查日志是否有 "异步模式" 字样
4. ✅ 运行测试脚本验证蓝牙延迟
