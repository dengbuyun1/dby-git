# 程序重构说明

## 📁 文件结构

重构后的代码结构更加清晰：

```
under_rasp/
├── main.py                      # 主程序（新增）
├── rasp_integration.py          # 系统整合模块（已重构）
├── motor_module_new2.py         # 电机控制模块
├── tcp_module_new.py            # TCP通信模块
├── algorithm_module.py          # 算法计算模块
├── lcd_module.py                # LCD显示模块
├── peripheral_module.py         # 外设控制模块
```

## 🔄 重构内容

### 1. **新增主程序 `main.py`**
   
   - **功能**: 程序入口，包含主函数和命令行参数处理
   - **职责**: 
     - 解析命令行参数（`--sim` 仿真模式）
     - 初始化日志系统
     - 创建 `RaspiIntegration` 实例
     - 设置信号处理器（Ctrl+C 优雅退出）
     - 调用系统初始化和启动
   
   **用法**:
   ```bash
   # 硬件模式（需要sudo权限）
   sudo python3 main.py
   
   # 仿真模式（无需硬件）
   python3 main.py --sim
   ```

### 2. **重构 `rasp_integration.py`**
   
   - **改动**: 移除主函数，保留 `RaspiIntegration` 类
   - **职责**: 纯粹的系统整合模块，可被导入使用
   - **导出类**: `RaspiIntegration`
   
   **导入方式**:
   ```python
   from rasp_integration import RaspiIntegration
   
   system = RaspiIntegration(simulation=False)
   system.initialize()
   system.start()
   system.shutdown()
   ```

## ✅ 优势

### 代码组织更清晰
- **main.py**: 执行入口，负责程序启动流程
- **rasp_integration.py**: 可复用的模块，可被其他程序导入

### 可维护性提升
- 主程序和初始化逻辑分离
- 便于单元测试（可直接导入 `RaspiIntegration` 类）
- 便于集成到其他项目

### 灵活性增强
- 可以创建多个不同的主程序（如调试版、生产版）
- 可以在其他Python脚本中复用 `RaspiIntegration`

## 📋 系统功能

主程序启动后会依次：

1. **初始化模块**
   - TCP服务器（端口5000）
   - 算法计算器
   - 电机控制器（TB6600）
   - LCD显示器（LCD1602）
   - LED指示器（RGB LED）
   - 外设控制器

2. **启动主循环**
   - 监听TCP数据
   - 实时计算胰岛素剂量
   - 驱动电机注射
   - 更新LCD显示
   - 控制LED状态

3. **优雅退出**
   - Ctrl+C 触发信号处理
   - 安全关闭所有模块
   - 清理GPIO资源
